Есть аналогия из заводом, где есть несколько конвееров по изготовлению часов.
Соответственно интерфейс WatchMaker отвечает на вопрос: Что изготовляем? -часы:
абстрактный метод  Watch createWatch(), который возвращает соответствующий класс часов

2 реализации производителя: DigitalWatchMaker и RomeWatchMaker
которые реализуют метод по созданию часов: Watch createWatch()

Описали интерфейс часов и 2 ее реализации. Под каждую реализацию часов нужно создать своего производителя часов и фабричный метод createWatch().

Гибкость заключается, что при добавлении нового вида часов, нам нужно будет реализовать производителя этих часов и сам класс часов.
А сам клиентский код: завод, произведение часов не меняется. добавляем подклассы.

Варианты    :
 интерфейс WatchMaker может быть абстрактным классом с реализованными некоторыми методами.
